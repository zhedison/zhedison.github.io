Sql题目训练笔记

1、group_concat([distant] 要连接的字段 [order by 排序字段][分隔符】) 该方法可以根据某个字段进行聚合，好像一对多那样。

2、concat() CONCAT 可以将多个字符串拼接在一起。

3、 LEFT(str, length) 函数 从左开始截取字符串，length 是截取的长度

4、 UPPER(str) 将字符串中所有字符转为大写

5、LOWER(str)将字符串中所有字符转为小写

6、 SUBSTRING(str, begin, end) 截取字符串，end 不写默认为空。

​ 注意：从1开始数。

7、group by 多个字段的时候用 ，（逗号）分割

8、UNION 可以将多个查询的结果合并在一起，类似与join,还有一个UNION ALL ，和UNION的区别就是，UNION ALL不去重，而UNION会去重。

9、if语句，if(判断条件，满足的返回，不满足返回)，如需多条件if,只能使用嵌套if,if(判断条件，满足返回，if(条件，满足，不满足))

10、写IN的时候，可以写多个字段。在查询存在多个最大值的时候，可以用IN来写。

11、对表根据某个字段进行查询时，分别进行排序并用union合并，比如：

SELECT name FROM student ORDER BY age

UNION

SELECT grade FROM student ORDER BY age; 这样是不可行的，会报错。

得这样才行

SELECT * from
(SELECT name FROM student
ORDER BY age limit 0, 1000) t1
UNION (all)
SELECT * from
(SELECT grade FROM student
ORDER BY age limit 0, 1000);

不加上limit也是不行的。

12、分组排名解决思路：比如找出每个部门前3高的工资（topn），可以用专用窗口函数rank,dense_rank,row_number，有什么区别呢？

select *,
rank() over (order by 成绩 desc) as ranking,
dense_rank() over (order by 成绩 desc) as dese_rank,
row_number() over (order by 成绩 desc) as row_num
from 班级;

分别查出来的是：

rank()是有并列名次的话会占用下一名次的位置，比如分数100，100，100，90，则排名为1，1，1，4

dense_rank（）则不会占用，排名为1，1，1，2

row_num()则不考虑排名并列的情况，1，2，3，4

dense_rank() over(partition by 班级 order by 成绩) 表示根据班级分组进行排名，partition by就表示分组。

所以说找出每个部门前3高的工资（topn）只需要根据部门分组进行排名，用where语句过滤出排名<=3的即可。

14、踩坑：在用 between … and … 判断时间存不存在这个区间的时候，要注意时分秒的边界问题。比如要找2020年内的时间，直接写 where time between “2020-01-01” and “2020-12-31” 这样就忽略了时分秒的问题，这默认的时分秒是0:0:0 也就是说如果有一个时间是2020-12-31 10:10:10这样就读不出来了。正确的做法应该是 between “2020-01-01 0:0:0” and “2020-12-31 23:59:59”。
