微服务之-网关与配置中心

一、网关

由于微服务的地址随时都有可能改变，不可能对外暴露这些接口，基于安全以及高内聚低耦合等设计，有必要做一个将外部系统和内部系统做一个切割。而且如果没有网关的话，权限需要每个服务都写一遍，大大增加代码量，导致jar包增大。另外如果没有网关服务的ip地址变了的话，调用了这个服务的服务都得改，增大工作量。

网关的实现主要有两个：

  • zuul
  • gateway

Zuul

Zuul 的功能：

  • 权限控制
  • 负载均衡
  • 动态路由
  • 静态资源处理
  • 监控

Zuul中的功能基本上都是在过滤器中实现的，过滤器的类型有:

  • Pre
  • Post
  • error
  • routing

GateWay

gateway也是一个网关，有以下功能：

  • 限流
  • 路径重写
  • 动态路由
  • 集成Spring cloud DiscoveryClient
  • 继承Hystrix断路器

和Zuul对比：

1、Zuul是Netflix公司的产品，GateWay是Spring家族的东西，可以更好的和Spring家族的其他组件进行整合。

2、Zuul不支持长连接，例如websocket

3、spring cloud gateway支持限流

4、Spring cloud gateway基于netty来开发，实现了异步和非阻塞，占用资源少，性能强于zuul

二、配置中心

因为微服务是由许许多多个服务组成的，它不像以前单机项目那样，只有一个配置文件，100个服务可能有100个配置文件，而且每个服务还有可能是集群部署，每个服务还分为生产环境、测试环境、开发环境，这样管理配置文件就成为了一个难题。因此，就出现了，Spring Cloud Config Server这样的工具，它可以从gitee、github或者数据库等，实时的获取配置文件，我们只需要把配置文件放在这些公共的地方，Spring Cloud Config Server就可以自动获取得到。
